# Security

## Reporting a vulnerability

Found a security issue? **Don't open a public issue.**

Email the maintainers directly or use GitHub's private vulnerability reporting.

We'll respond within 48 hours and work with you to fix it.

## What we consider security issues

- Authentication bypass
- Credential leaks
- Remote code execution
- Anything that could expose user data

## What's NOT a security issue

- Bugs that require physical access to the machine
- Theoretical attacks with no practical exploit
- Issues in dependencies (report those upstream)
